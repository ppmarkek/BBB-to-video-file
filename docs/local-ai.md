# Локальная обработка лекций

Konspekt создаёт материалы лекции на компьютере пользователя. Для базовой работы не нужны ключи OpenAI, DeepSeek или другого API.

## Установка

Из корня проекта выполни:

```powershell
.\scripts\setup_local_ai.ps1
```

Сценарий создаёт `.venv` и устанавливает пакет Konspekt с локальным Faster-Whisper.

## Что происходит при подготовке

1. Загружается дорожка преподавателя из публичной BBB-записи.
2. FFmpeg извлекает моно-аудио 16 кГц.
3. Faster-Whisper локально создаёт `transcript.md` и `transcript.json`.
4. Если в записи есть демонстрация экрана, сохраняются кадры с выбранным в настройках интервалом (по умолчанию 60 секунд).
5. При наличии Tesseract создаётся `screen-notes.json` с OCR-текстом.

Прогресс в приложении показывает реальные этапы, прошедшее время и последнее действие. Во время распознавания речи процент обновляется по таймкодам Whisper, во время OCR — по числу обработанных кадров. Скорость зависит от длительности записи и компьютера.

Если обработка прервалась, нажми «Повторить». Уже скачанные медиа, извлечённое аудио, готовая транскрипция, кадры и OCR-заметки используются повторно. Например, ошибка на OCR не запускает заново транскрибацию всей лекции.

## Скорость и качество

В разделе «Настройки» доступны три локальные модели:

- `tiny` — самая быстрая, подходит для черновой расшифровки;
- `base` — рекомендуемый баланс скорости и качества;
- `small` — точнее на сложной речи, но заметно медленнее на CPU.

Там же можно увеличить интервал кадров или полностью отключить обработку демонстрации экрана (кадры и OCR). Это не влияет на аудио и транскрипцию и ускоряет обработку длинных записей.

## OCR с Tesseract

Установка для Windows:

```powershell
winget install UB-Mannheim.TesseractOCR
```

Если Tesseract отсутствует, Konspekt всё равно создаст транскрипцию и кадры, но честно отметит пропуск OCR. Если обработка экрана отключена в настройках, видео демонстрации не скачивается.

Ошибка распознавания одного кадра не останавливает всю лекцию: кадр пропускается, а остальные материалы сохраняются.

## Необязательное API

OpenAI и DeepSeek API нужны только для автоматического создания готового `lesson.md`. Локальная подготовка работает без ключа. При API-вызове отправляется только текстовый пакет без BBB-ссылки и meeting ID; аудио, видео и изображения не отправляются.

API остаётся отдельным способом подключения: вход через личный ChatGPT не создаёт API-ключ, не переносит подписку ChatGPT в Platform API и не меняет API-биллинг.

## Личный ChatGPT через Codex

Для работы без API-ключа Konspekt может использовать личный ChatGPT через локально установленный Codex CLI:

1. Установи Codex CLI и проверь `codex --version` в PowerShell.
2. В Konspekt выбери вход через личный ChatGPT.
3. В отдельном окне Microsoft Edge WebView2 вручную заверши авторизацию OpenAI.
4. Выбери одну из моделей Codex, которые app-server вернул для этого аккаунта.
5. Konspekt передаст Codex только текстовый пакет и автоматически сохранит результат как `lesson.md`.

Интеграция использует официальный [Codex app-server](https://learn.chatgpt.com/docs/app-server), но этот интерфейс помечен как Experimental и может меняться вместе с Codex CLI. Это не общий OAuth к обычному ChatGPT API: приложение не обещает наличие привычных моделей из интерфейса ChatGPT, а показывает только доступный аккаунту каталог Codex.

WebView2 используется только как одно ручное окно авторизации. Konspekt не открывает внутри себя чат `chatgpt.com`, не автоматизирует DOM, не извлекает ответы из веб-страницы и не читает cookies. Документация app-server описывает управляемый browser login, но отдельно не гарантирует совместимость embedded WebView; поэтому для этого Experimental-пути нужны актуальные Codex CLI и Microsoft Edge WebView2 Runtime.

## Где лежат данные

```text
%LOCALAPPDATA%\Konspekt\lectures\<meeting-id>\
```

Там сохраняются `audio.wav`, транскрипция, кадры, OCR-заметки, пакет контекста и готовый `lesson.md`.
